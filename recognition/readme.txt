
1. 打开main_recogonsegnoidnot558_batch_20210217并执行
2. 程序将遍历input_noID文件夹下所有以.ano.ano.txt结尾的文件并执行识别
	修改20和21行，可选择执行 input_noID或 input_withID文件夹下的文件
3. input_noID文件夹下有三个测试文件（该文件夹下所有文件均未提供手工识别结果）：
	第一个有558个完整细胞
	第二个从第一个里面删掉了7个（有551个有效细胞）
	第三个在第二个文件的基础上，手工固定了2个细胞的名称→通过将ano.ano.txt中相应细胞的comment一栏由之前的*MANUAL*修改为*FIX*即可→输出文件中相应细胞的fix状态保持，以便于循环修正并识别
	识别结果以相同文件名输出在output文件夹
4. input_withID文件夹下的文件命名方式同input_noID，区别在于均提供了完整的手工识别结果，以便于测试对比精度
	输出在output文件夹下输出识别结果之外，还输出了recogReport报告，以便于分析识别情况
5. output文件夹下存放识别结果
	注意：识别结果中增加了最后一列，以用于存放识别置信度→因为数值较为相近，个人感觉用处不太大！
